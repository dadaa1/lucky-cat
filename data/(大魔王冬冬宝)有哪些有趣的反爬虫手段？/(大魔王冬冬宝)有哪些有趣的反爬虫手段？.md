做数据这行,爬虫和反爬两边都待过。两边打了这么多年,这篇把它按四大梯队梳理出来,一共二十多种,从最经典的前端障眼法,到 2025 到 2026 年冒出来的反 AI 爬虫新玩法,能用的都标了工具和链接。

筛选标准得先讲清楚,免得清单注水。我只收三类:一是"看着离谱但真在生产环境跑"的,像把整页数字切成雪碧图这种;二是这两年刚出现、还在快速演化的,比如工作量证明和 AI 迷宫;三是有明确破解与对抗资料、你以后动手时能顺着往下查的。纯理论、纯 PPT 的招我一律没收。

第一梯队:客户端障眼法。这一层的思路很统一,数据照样发到浏览器,但发出来的不是你以为的那个东西。人眼看着正常,爬虫拿到手是一堆乱码或者错的。

1. 字体反爬。网站塞一个自定义的 WOFF 字体文件,把数字 0 到 9 重新映射。页面源码里写的是一串乱码字符,浏览器用那个字体渲染出来才是真实数字。猫眼电影的票房、58 同城的电话号码都用过这招。最阴的是猫眼,不同时间刷新会返回不同的字体文件,同一个字符在这次是 3,下次可能是 7,你得抓好几份字体文件找共性才能定位真实映射。破解一般靠 fontTools 库解析字形轮廓,再配 OCR 识别。中文社区里静觅的那篇"我终于弄懂了字体反爬是个啥玩意"讲得最透,想入门可以从那看。

2. SVG 坐标映射。这是大众点评的招,比字体反爬还绕一层。它把文字画进一张 SVG 图里,页面上每个字符对应一组 x、y 偏移坐标,你得从 CSS 里把 SVG 抠出来,按坐标算出这个字符落在图里哪个位置,才能还原。等于把"查字典"变成了"查地图"。大众点评这些年字体库和 SVG 偏移两套换着用,爬的人叫苦不迭。

3. CSS 伪元素塞数据。关键内容不写在标签里,而是塞进 ::before 或者 ::after 的 content 属性。你抓 HTML 抓不到,得去解析样式表。价格、评分这种字段最爱这么玩,因为短。

4. CSS 偏移打乱顺序。源码里数字的顺序是乱的,比如真实价格 1234,HTML 里可能按 3142 的顺序写,再用负 margin 和绝对定位把每一位在视觉上摆回正确位置。你按 DOM 顺序读是错的,按屏幕坐标读才对。去哪儿网早期干过。

5. 雪碧图。把 0 到 9 十个数字拼成一张大图,页面上用 background-position 去截取显示哪一位。你抓到的就是一串 background-position 的像素值,得反查每个偏移量对应哪个数字。等于把文本彻底变成了图片切片。

6. 整块渲染成图片。比 5 更暴力,直接把关键数据用 canvas 画出来,或者服务端生成图片返回。没有文本层,爬虫只能上 OCR。成本高,所以一般只用在最值钱的字段上,比如财报数字、核心报价。

7. 蜜罐链接。页面里埋 display:none 或者移出屏幕的隐藏链接,真人看不见也不会点,爬虫顺着 a 标签一把梭就会踩中。一踩中,风控直接拉黑这个 IP。这招几乎零成本,是老牌反爬里性价比最高的。

8. 假数据投毒。检测到你是爬虫之后,不封你,反而给你返回一份看着正常、实则错误的数据。价格给你改几块钱,库存给你写个假的。这招最恶心的地方在于你根本不知道自己被投毒了,数据入库了才发现全是脏的。r/Blogging 上有站长讨论过这个思路,有人的原话是与其一个个封,不如给可疑流量限速加喂假数据,反正想彻底防住抓取几乎不可能。

第二梯队:请求指纹层。前端障眼法拼的是解析,这一层拼的是伪装。你还没拿到页面,握手阶段就被认出来了。

9. TLS 指纹,也就是 JA3 和 JA4。这是 2026 年最主流、也最多人栽跟头的一招。你的客户端发起 HTTPS 连接时,TLS 握手里的加密套件、扩展项、排列顺序会拼成一个指纹。Python 的 requests、Go 的默认 client,它们的握手长相和真实 Chrome 差得远,Cloudflare、DataDome、Akamai 这些一眼就认出来。Scrapfly 那篇"How Cloudflare Detects Bots"专门拆过这块,说很多教程到今天都还不讲 TLS 这一层,导致大量人换了 UA、加了 header 还是被封,根本没意识到问题出在握手上。对抗工具是 curl-impersonate,GitHub 地址 [github.com/lwthiker/curl-impersonate,它给](http://github.com/lwthiker/curl-impersonate,%E5%AE%83%E7%BB%99) libcurl 打补丁,把加密套件、扩展顺序、GREASE 值全仿成 Chrome 或 Firefox 的样子。Python 用它的 cffi 绑定 curl\_cffi 更方便,地址 [github.com/lexiforest/curl\_cffi。补一个细节,Chrome](http://github.com/lexiforest/curl_cffi%E3%80%82%E8%A1%A5%E4%B8%80%E4%B8%AA%E7%BB%86%E8%8A%82,Chrome) 每次握手会往套件里随机塞 GREASE 值,所以 JA3 哈希每次都变,想要稳定指纹得匹配 JA4,因为 JA4 是先排序再哈希的。

10. HTTP/2 指纹。TLS 之后还有一层。HTTP/2 的握手里,SETTINGS 帧的参数、帧发送的顺序、窗口大小,不同客户端也不一样。你 TLS 仿得再像,HTTP/2 层露馅照样被抓。这也是为什么光换 requests 的 header 没用。

11. Header 顺序与一致性。真实浏览器发的请求头,顺序是固定的,而且各字段之间会相互印证,比如 UA 说自己是 Chrome,那 sec-ch-ua 这些客户端提示头就得对得上。爬虫库经常头字段齐了但顺序错了,或者 UA 和其他特征打架,风控做个一致性校验就露了。

12. 自动化特征检测。navigator.webdriver 这个属性,被 Selenium、Playwright 驱动的浏览器默认是 true。再进阶的会检测 CDP,也就是 Chrome DevTools Protocol 的连接痕迹,以及一堆 headless 浏览器特有的 JS 环境差异。你用无头浏览器,不做隐身处理,进门就被记了一笔。

13. 浏览器指纹。Canvas 指纹、WebGL 指纹、字体列表、屏幕分辨率、时区,这些拼起来能给每台设备算一个近乎唯一的 ID。Cloudflare 会综合看这些,加上 TLS 一起算一个信任分。你换 IP 没用,指纹没变还是同一个你。

第三梯队:行为与风控层。前两层是静态的,这一层是动态的,它不看你像不像,它看你干得像不像人。

14. 鼠标轨迹与行为生物特征。DataDome 这类厂商会采集鼠标移动的曲线、点击的节奏、页面停留和滚动的模式。真人的鼠标是有加速度、有抖动、走贝塞尔曲线的,脚本要么直线瞬移,要么完全不动。行为对不上,后台直接标记。DataDome 号称针对每个客户跑超过八万五千个定制机器学习模型,等于每个网站的正常流量长什么样,它都单独学过一遍。

15. 速率异常与自适应阈值。Cloudflare 现在不玩固定阈值了,它给每个域名学一套什么叫正常,再动态定门槛。你在一个访问量本来就低的站上突然高频请求,哪怕绝对量不大,相对这个站的基线也异常,一样触发。

16. 住宅代理识别。很多爬虫用住宅代理来伪装成真实用户,风控就反过来识别代理特征。r/Blogging 上有站长的原话是,他们发现的爬虫用住宅 VPN,几乎和真实用户没法区分。另一个站长说他干脆不硬刚了,改去扫 SSL 日志看那些机器人农场到底想干嘛。这一条其实说明了反爬的真实困境,防到最后你可能会误伤真人。

17. 无感验证。Cloudflare Turnstile、各家的滑块和点选,现在越来越多做成无感的,不弹框,后台悄悄跑一段 JS 挑战验证你是不是浏览器环境。你要是纯 HTTP 客户端,连挑战都执行不了,直接卡在门口。

18. 蜜标。给不同用户或者不同渠道的数据里,埋进只有你自己知道的假记录,或者微调过的水印。等这份数据在别处出现了,一比对蜜标,立马知道是从哪泄漏、被谁抓走的。这招不拦你,专门用来抓现行和打官司。

第四梯队:反 AI 爬虫的新玩法。这是 2025 到 2026 年最热闹的一块。大模型公司的爬虫不讲武德,无视 robots.txt,把很多开源项目的服务器刷到崩,于是一批带梗又硬核的新工具冒出来了。

19. Anubis 工作量证明。这是我个人觉得最有意思的一个。作者 Xe Iaso 因为亚马逊的爬虫把自己的 Git 服务器刷爆,一怒之下写了 Anubis。它在你访问网站前先塞一道 SHA-256 的工作量证明题,算起来费劲、验起来很快,真人等一两秒无感,爬虫要大规模抓就得付出成倍的算力成本。名字取自古埃及那个称量灵魂的死神,作者说 Anubis 称量的是每个 HTTP 请求的灵魂。加载页面还配了个二次元少女形象的看门狗做吉祥物。2025 年 1 月发布,现在 GNOME 的 GitLab、Linux 内核邮件列表存档、FFmpeg、Wine、FreeCAD、甚至联合国教科文组织都在用。TechCrunch 专门写过一篇"开源开发者用巧劲和怒火对抗 AI 爬虫",The Register 那篇标题更直接,叫"Anubis 抵御 LLM 爬虫大军"。有个现实的注脚,Codeberg 在 2025 年 8 月反馈说不少爬虫已经学会解 Anubis 的题了,不过它扛了好几个月还是值的。攻防就是这样,没有一劳永逸。

20. Cloudflare AI 迷宫。检测到不守规矩的 AI 爬虫,Cloudflare 不封它,反手给它喂一串 AI 生成的、看着挺真实其实全是废话的假页面,页面里再埋一堆隐藏链接,把爬虫诱进一个无限延伸的迷宫里空转,烧它的算力。逻辑很妙,正常人不可能顺着一堆生成垃圾往里点四五层,谁点进去谁就是机器人,于是这套迷宫同时又变成了新的蜜罐和指纹工具。官方博客说这套东西已经服务于每天五百亿次爬虫请求。它对 SEO 和真实用户零影响,只有坏爬虫看得见。

21. Nepenthes 焦油坑。AI 迷宫的思路其实是 Nepenthes 这类焦油坑工具先带起来的。焦油坑把爬虫困在一个无限循环的机器生成垃圾里,一边耗它的算力,一边往它的训练集里灌毒。等于你抓我,我就往你的模型里投脏数据。这类工具在开源圈很受欢迎,带着一股鱼死网破的爽感。

22. 默认拦截加按爬付费。Cloudflare 从 2025 年 7 月起,默认帮客户拦截 AI 爬虫,还在推按抓取次数付费的模式,想抓我的内容训练模型可以,拿钱来。这是把反爬从技术对抗,往商业规则那边推了一步,可能是接下来两三年最值得盯的方向。这一块展开讲太长,水也深,我打算下次单开一篇聊数据版权和按爬付费这盘棋。

写到这我自己的一个判断是,反爬没有银弹。第一梯队的前端障眼法,解析成本高但迟早被逆向;第二梯队的指纹层,curl-impersonate 这类工具一出就被追平;第三梯队的行为风控最难绕,但也最容易误伤真人;第四梯队的反 AI 玩法最新最带感,可 Anubis 被学会解题也就半年。真正扛得住的从来是组合拳加持续迭代,单靠哪一招都撑不久。字体反爬那套破解实战我手上有完整代码和踩坑记录,涉及 fontTools 加坐标绘制加 OCR 那一整条链路,展开太长,留着下次单独写一篇。
